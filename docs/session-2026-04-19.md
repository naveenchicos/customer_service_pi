## What Was Built

### Application Layer (`src/`)

| File | Purpose |
|---|---|
| `src/config.py` | Pydantic Settings — validates all env vars at startup, crashes fast on missing required vars. `SecretStr` used for `API_KEY` so it never appears in logs. |
| `src/db.py` | Async SQLAlchemy engine + `SessionLocal` + `get_db()` FastAPI dependency. `pool_pre_ping=True` handles Cloud SQL connection resets. |
| `src/models/account.py` | `Account` ORM model — UUID primary key, soft-delete via `status`, composite indexes for search patterns, audit fields (`created_by`, `updated_by`). |
| `src/schemas/account.py` | Pydantic schemas: `AccountCreate`, `AccountUpdate`, `AccountResponse`, `AccountSummary`, `PaginatedAccounts`, `ErrorDetail`. Each operation has its own schema so shapes never bleed into each other. |
| `src/middleware/correlation_id.py` | Reads or generates `X-Correlation-ID` on every request. Validates externally-provided IDs are valid UUIDs to prevent log injection. |
| `src/middleware/security_headers.py` | Adds OWASP A05 headers: `CSP`, `HSTS`, `X-Frame-Options`, `X-Content-Type-Options`, `Cache-Control: no-store`, `Server` masking. |
| `src/middleware/api_key_auth.py` | OWASP A07 — validates `X-API-Key` header using `hmac.compare_digest` (timing-safe). `/health`, `/docs`, `/openapi.json` are exempt. |
| `src/clients/customer_client.py` | httpx async client with 500ms timeout, `@circuit` breaker (5 failures / 30s recovery), `get_customer_or_none()` fallback that returns `None` instead of raising when circuit is open. |
| `src/services/account_service.py` | All business logic: create (email normalised to lowercase), get by ID, get by customer number, paginated search (ILIKE, capped at 100), patch (true partial update), soft-delete. |
| `src/routers/accounts.py` | 6 REST endpoints with full OpenAPI descriptions, examples, and per-status-code response schemas. |
| `src/routers/health.py` | `GET /health` (liveness, no auth) + `GET /health/dependencies` (circuit breaker state per downstream). |
| `src/main.py` | App factory, lifespan wiring (init/close clients), middleware stack, custom `ValidationError` handler (never exposes raw Pydantic errors — OWASP A05), catch-all exception handler (logs full trace internally, returns safe generic message). |

### OWASP Controls Implemented

| Control | Implementation |
|---|---|
| A01 Broken Access Control | UUID PKs — no sequential IDs guessable externally. Soft-delete preserves audit trail. |
| A03 Injection | Pydantic validators: regex on `customer_number` and `phone`, control-char check on names, `EmailStr` validation. SQLAlchemy ORM parameterised queries — no raw SQL. ILIKE search uses bound parameters. |
| A05 Security Misconfiguration | `SecurityHeadersMiddleware` on every response. `/docs` and `/openapi.json` disabled in production. Non-root user (UID 1001) in Dockerfile. `readOnlyRootFilesystem: true` in K8s deployment. Stack traces never returned in responses. |
| A07 Auth Failures | `ApiKeyAuthMiddleware` with `hmac.compare_digest`. `SecretStr` prevents key leaking in repr/logs. `WWW-Authenticate` header returned on 401. |
| A09 Logging Failures | Structured JSON logs via `python-json-logger`. `X-Correlation-ID` on every request. Full tracebacks logged server-side only. |

## Issues Encountered and Resolutions

### 1. pydantic-core has no wheel for Python 3.14 with pinned version

**Symptom:** `pip install -r requirements.txt` failed building `pydantic-core` from source.

**Root cause:** `pydantic==2.10.3` requires `pydantic-core~=2.27` which did not have a pre-built wheel for Python 3.14. Building from source requires Rust + maturin and failed.

**Fix:** Upgraded to `pydantic>=2.13.1` and `pydantic-settings>=2.13.1` which ship with `pydantic-core==2.46.1` — this version has a CPython 3.14 wheel (`cp314-cp314-macosx`).

---

### 2. SQLAlchemy `Mapped[X | None]` breaks on Python 3.14

**Symptom:** All tests failed at collection time with:
```
TypeError: descriptor '__getitem__' requires a 'typing.Union' object but received a 'tuple'
```

**Root cause:** Python 3.14 changed how `Union.__getitem__` works. SQLAlchemy re-evaluates `Mapped` annotations using `get_type_hints()`, which resolves string annotations and then calls `Union.__getitem__(types)` — but in Python 3.14, `Union` is no longer subscriptable with a tuple this way.

**Attempted fix that did NOT work:** Adding `from __future__ import annotations` to defer evaluation — SQLAlchemy still re-evaluates via `get_type_hints()`.

**Fix that worked:** Replaced all `Mapped[str | None]` with `Mapped[Optional[str]]` using `from typing import Optional` in `src/models/account.py`. Also upgraded SQLAlchemy from `2.0.36` to `2.0.49`.

**CLAUDE.md note added:** `Mapped[Optional[X]]` must be used in all ORM models — never `Mapped[X | None]` — on Python 3.14.

### 3. pytest-asyncio requires explicit mode configuration

**Symptom:** Tests decorated with `@pytest.mark.asyncio` were collected but not run; deprecation warnings about `asyncio_default_fixture_loop_scope`.

**Fix:** Added `pytest.ini`:
```ini
[pytest]
asyncio_mode = auto
asyncio_default_fixture_loop_scope = function
```

**Root cause:** `cloudresourcemanager.googleapis.com` was not enabled on the project, and the service account didn't have permission to enable it.

### 8. `.env` file contained shell commands, not environment variables

**Symptom:** `python-dotenv` printed warnings: `could not parse statement starting at line 1/2`. App still started because `python-dotenv` skipped invalid lines, but env vars were not loaded from the file.

**Root cause:** The `.env` file originally contained:
```
gcloud auth application-default login
gcloud config set project pi-dev-ai
```
These are shell commands, not `KEY=VALUE` pairs.

**Fix:** Replaced the entire file with valid `KEY=VALUE` pairs including `DATABASE_URL`, `API_KEY`, `ENVIRONMENT`, etc.

## Local Development Setup (Cloud-Connected)

The service runs locally but uses the real Cloud SQL instance in GCP. No local database needed.

```
[FastAPI on localhost:8000]
        │
        ▼
[Cloud SQL Auth Proxy on localhost:5432]
        │  (encrypted IAM-authenticated tunnel)
        ▼
[Cloud SQL PostgreSQL — pi-dev-ai-493823:us-central1:py-dev-ai]
```

### Prerequisites

- `brew install --cask google-cloud-sdk`
- `brew install cloud-sql-proxy`
- Service account key: `pi-dev-ai-493823-3001ad400aed.json`
- `.env` file with `DATABASE_URL`, `API_KEY`, `ENVIRONMENT=local`

### Start the service

```bash
# Terminal 1 — Auth Proxy (keep running)
gcloud auth activate-service-account \
  --key-file="pi-dev-ai-493823-3001ad400aed.json"
cloud-sql-proxy "pi-dev-ai-493823:us-central1:py-dev-ai" --port=5432

# Terminal 2 — App
source .venv/bin/activate
alembic upgrade head       # first time only, or after new migrations
uvicorn src.main:app --reload
```

Service: `http://localhost:8000`
Interactive API docs: `http://localhost:8000/docs`
