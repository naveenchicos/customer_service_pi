### 2. CD workflow â€” release/* trigger + auto merge-to-main

**Before:** Triggered on `push: branches: ["main"]`. Four jobs: test, build-push, deploy-production, deploy-staging.

**After:** Triggered on `push: branches: ["release/*"]`. Four jobs: test, build-push, deploy-production, merge-to-main.

Key changes:
- Trigger changed from `main` to `release/*`
- `deploy-production` environment gate (manual approval) kept
- New job `merge-to-main` runs after successful deploy:
  - Extracts version from branch name (`release/v1.1.0` â†’ `v1.1.0`)
  - Merges release branch into `main` with `--no-ff`
  - Creates version tag and pushes

```yaml
- name: Extract version from branch name
  id: version
  run: |
    BRANCH="${GITHUB_REF#refs/heads/}"
    VERSION="${BRANCH#release/}"
    echo "branch=$BRANCH" >> $GITHUB_OUTPUT
    echo "version=$VERSION" >> $GITHUB_OUTPUT

- name: Merge release branch into main
  run: |
    git config user.name "github-actions[bot]"
    git config user.email "github-actions[bot]@users.noreply.github.com"
    git checkout main
    git merge --no-ff origin/${{ steps.version.outputs.branch }} \
      -m "chore: merge ${{ steps.version.outputs.branch }} into main"
    git push origin main

- name: Create version tag on main
  run: |
    git tag ${{ steps.version.outputs.version }}
    git push origin ${{ steps.version.outputs.version }}
```

Staging job removed (staging branch is no longer part of the workflow).

File: `.github/workflows/cd.yaml`

## Conflict Behaviour â€” Important Note

If `merge-to-main` encounters a conflict, the job **fails** but the GKE deploy has already succeeded. In that case, merge and tag manually:

```bash
git checkout main && git pull
git checkout release/vX.Y.Z
git merge main        # resolve conflicts
git push origin release/vX.Y.Z

git checkout main
git merge --no-ff release/vX.Y.Z
git push origin main
git tag vX.Y.Z && git push origin vX.Y.Z
```

**Prevention:** Always back-merge hotfixes into `develop` immediately. Conflicts on `main` only happen when a hotfix was applied to `main` but not back-ported.
